    # 内置函数

    # 1. JS 数据类型有哪些？

    JS 一共有8种数据类型：undefined、null、boolean、string、number、bigInt、symbol、object 其中 JS 数据类型可分为基础数据类型和引用数据类型。

    基础数据类型有：undefined、null、boolean、string、number、bigInt、symbol 基本数据类型存储在栈内存中

    引用数据类型：Object => (Object 子类型)内置函数：Array、Function、RegExp、Date 引用数据类型存储在堆内存中

    # 2. 基本包装类型（强类型转换）

    基本包装类型分别有：Boolean、Number、parseInt、parseFloat、String

    1. Boolean

    规则：

    • 转换为 false: false、""、0、NaN、null、undefined，其他都为 true
    • 判断 NaN：isNaN(NaN) // true，其他都为 false

    2. Number

    规则：

      1. 布尔值，true 转换为1，false 转换为 0；
      1. 数值，直接返回；
      1. null，返回 0；
      1. undefined，返回 NaN；
      1. 字符串，如果字符串是数值，忽略前面的0，转换为十进制，如果字符串是浮点值，则转换为响应的浮点值。如果字符串是十六进制格式，则将其转换为十进制；如果是空字符串，则返回0，其他情况，返回 NaN
      1. 对象，调用 valueOf()方法，并按照上述规则转换返回的值。如果转换结果是 NaN，则调用toString()方法，再按照字符串规则转换

    3. parseInt

    更专注于字符串是否包含数值模式。 字符串最前面的空格会被忽略，从第一个非空字符串开始转换。 如果第一个字符不是数值字符、加号和减号，parseInt() 即刻返回 NaN

    1. parseInt("1234blue") // 1234
2. parseInt("") // NaN
3. parseInt("0xA") // 10
4. parseInt("22.5") // 22

    4. parseFloat

    1. parseFloat("1234blue") // 1234
2. parseFloat("0xA") // 0
3. parseFloat("22.5") // 22.5
4. parseFloat("22.34.5") // 22.34
5. parseFloat("0908.5") // 908.5
6. parseFloat("3.125e7") // 31250000

    5. String

      1. 如果有 toString() 方法。则调用该方法
      1. 如果值是 null，返回 "null"
      1. 如果值是 undefined，返回 "undefined"

    # 3. typeof null === 'object'

    不同的对象在底层都表示一个二进制，在 JavaScript 中二进制前三位都为0，会被判别为 Object，而 null 的二进制表示全 0，自然前三位也是 0，所以 typeof null === 'object'

    • 1：整型（int）
    • 000：引用类型（object）
    • 010：双精度浮点型（double）
    • 100：字符串（string）
    • 110：布尔型（boolean）

    # ES6

    # 1. 什么是 ES Module ？

    ES Module 是 ES6 推出的静态导出方式，在ES Module中导出分为两种，单个导出(export)、默认导出(export default)，单个导出可以导出我想要的值。
    那么默认导出就是全部直接导出。ES Module使用的是import/export语法进行导入导出。如果要单个导入则必须使用花括号{ }，如果全部导入则不用。

    # 2. ES Module 解决了什么问题？

    ES Module 解决了变量污染问题，依赖顺序问题，ES Module语法也是更加灵活，导出值也都是导出的引用，导出变量是可读状态，这加强了代码可读性。

    # 3. ESModule 和 ComｍonJS 有什么不同？

    CommonJS

    • CommonJS可以动态加载语句，代码发生在运行时
    • CommonJS混合导出，还是一种语法，只不过不用声明前面对象而已，当我导出引用对象时之前的导出就被覆盖了
    • CommonJS导出值是拷贝，可以修改导出的值，这在代码出错时，不好排查引起变量污染

    ES Module

    • ES Module是静态的，不可以动态加载语句，只能声明在该文件的最顶部，代码发生在编译时
    • ES Module混合导出，单个导出，默认导出，完全互不影响
    • ES Module导出是引用值之前都存在映射关系，并且值都是可读的，不能修改

    # 4. ES6 的新特性

    let、const、import/export、new Set、class(创建类)、展开运算符(...arr)、模板字符串(${})、 解构赋值、for of 、箭头函数(()=>{})、symbol(唯一的值)、数组新增的方法（flat、find、findIndex） 对象新增方法（Object.assign()、Object.values()、Object.keys()、Object.create()...）

    # 5. var、let、const 的区别？

      1. let 和 const 声明变量时不存在变量提升，即如果需要使用该变量，需要声明后使用
      1. let 和 const 不能重复声明变量，如果重复声明会报错
      1. let 和 const 在全局声明变量不会给 window 增加属性
      1. let 和 const 出现在代码块中，会把代码块变成块级作用域，并且出现暂时性死区

    # 6. 箭头函数和普通函数的区别？

      1. 箭头函数是匿名函数，不能作为构造函数来使用，不能 new
      1. 箭头函数没有原型属性（prototype）
      1. this 指向不同，箭头函数指向最靠近的函数的 this
      1. 不可以使用 arguments 对象，该函数体内不存在

    # 7. 什么时 itarator? 默认部署了iterator 的方法有哪些?

    对于可迭代的数据解构，ES6在内部部署了一个 [Symbol.iterator] 属性，它是一个函数，执行后会返回iterator对象 （也叫迭代器对象），而生成iterator对象 [Symbol.iterator] 属性叫iterator接口,有这个接口的数据结构即被视为可迭代的

    部署的方法有：
    Array、Map、Set、String、TypedArray（类数组）、函数的 arguments 对象、NodeList 对象

    # 1. 介绍一下 JavaScript 的执行上下文和执行栈

    # 2. 介绍一下 JavaScript 的作用域和作用域链

    2.1 作用域
    作用域是源代码中定义变量的区域，JavaScript 采用的是词法作用域，也就是静态作用域。（函数在定义的时候就已经决定了）

    2.2 作用域链
    作用域链在查找变量的时候会从当前的上下文的变量对象中查找，如果没有找到，则会从父级执行上下文中查找，一直找到全局上下文的变量对象，也就是全局对象，这样由多个执行上下文的变量对象构成的链表叫做作用域链。

    # 3. 介绍一下 JavaScript 的闭包以及已经运用的场景

    • 垃圾回收
    • 避免全局变量
    • 闭包是一个可以访问外部作用域的内部函数，而闭包的外部作用域是在其函数定义的时候就已经决定了，而不是在函数调用的时候。

    # 原型/继承

    # 1. 介绍一下 JavaScript 的原型

    原型其实就是一个特殊的对象，在声明函数的时候就已经创建的，原型链是通过 proto 来实现的。

    # 2. 原型链是什么

    当对象查找一个属性的时候，如果没有在自身找到，则会查找自身的原型，如果原型还没有找到，那么会继续查找原型的原型，直到找到 Object.prototype 的原型时，此时原型为 null，查找停止。 这种通过原型链接的逐级向上的查找链被称为原型链

    # 3. 什么是继承

    如果一个对象可以使用另外一个对象的属性或者方法就叫继承

    # 4. 如何实现原型继承(6种+ES6继承)

    ES5继承方式

    • 1.原型链继承（new）
    • 2.构造函数继承（call）
    • 3.组合继承（new + call + constructor）
    • 4.原型式继承（Object.create()）
    • 5.寄生式继承（Object.create + 原型方法）
    • 6.组合寄生式继承（Object.create() + call + constructor）

    # 4. JavsScript 的异步解决方案

    回调函数

    • 缺点：回调地狱、不能用 try / catch、不能 return
    • 优点：通过异步解决同步的问题

    Promise

    • 优点：解决了回调地狱的问题

    • 缺点：await 将异步代码改造成同步代码，如果代码中没有适当的使用，会造成性能的下降
    • 优点：JS 异步编程的终极解决方案，将异步执行用同步的方式表示出来代码清晰

    # 节流与防抖

    # 1. 介绍一下什么是节流和防抖

    • 防抖：高频事件触发后，n 秒后只会执行一次，如果高频事件再次被触发，则重新计算事件
    • 节流：高频事件触发后，n 秒内只会执行一次，所以节流会稀释函数的执行频率

    # 2. 如何实现节流和防抖

    • 手写一种防抖
    • 手写三种节流并搞懂分别优缺点

    # 3. 防抖和节流的相同点和不同点

    • 相同点：都可以用 setTimeout 实现，都是降低回调的执行频率，节省计算资源
    • 不同点：防抖是一段时间后执行函数，节流是一段时间内只执行函数一次，节流是提高性能

    # 4. 介绍一下节流和防抖的运用场景

    • 节流：滚动加载、resize 窗口大小
    • 防抖：input 搜索框搜索输入、手机号、验证码检测

    # 事件机制 / Event Loop

    # 1. 什么是单线程？

    JavaScript 语言的一大特点就是单线程，就是在特定的时刻只有特定的代码能够被执行，并阻塞其他代码的执行，也就是说同一时间只能做一件事

    # 2. 介绍一下事件循环

    事件循环是指浏览器或者 Node 的一种解决 JavaScript 单线程运行时不会阻塞的一种机制，也就是我们经常使用的异步的原理。JavaScript 单线程分为同步任务和异步任务，在主程序中的所有同步代码执行完毕，然后再读取任务队列中的异步任务，而异步任务又分为宏任务和微任务，执行一个宏任务，尝试清空微任务，这个过程是不断循环的，所以整个的运行机制叫做事件循环(Event Loop)。

    # 3. 为什么 JS 有事件循环机制

    JavsScript 从诞生起就是单线程，但单线程导致很多任务需要排队，只有一个任务执行完毕才能执行下一个任务，但如果一个任务执行时间过长，就会导致阻塞，所以 JavaScript 引入事件循环机制。

    # 4. 只有宏任务不行？为什么引入微任务

    因为宏任务是秉承着先进先出的原则，在JS 代码中如果需要一个优先级比较高的先执行，但是宏任务是先进先出的原则，所以必须引入微任务，让需要优先执行的代码优先执行。

    # 5. Node 中的事件循环和浏览器中的事件循环有什么区别？

    在 Node V10 后的版本跟浏览器保持一致

    # 6. 宏任务与微任务有什么区别

    • 宏任务：整个代码，setTimeout、setInterval、setImmedia
    • 微任务：Promise、async / await，process.nextTick、Object.observe、MutationObserve
